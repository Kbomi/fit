# 체질 기반 생활 가이드 서비스 기획서

작성일: 2026-06-29

## 1. 서비스 개요

### 서비스 한 줄 설명
사상체질 테스트를 통해 사용자의 체질을 확인하고, 체질별 음식, 생활 루틴, 컨디션 관리 정보를 개인화해 제공하는 모바일 중심 웹 서비스.

### 핵심 가치
사용자는 자신의 체질을 재미있고 쉽게 확인한 뒤, 결과를 단순히 읽고 끝내는 것이 아니라 오늘 먹을 음식, 피로 관리, 운동, 수면, 계절별 관리에 바로 참고할 수 있다.

### 초기 제품 형태
초기에는 앱보다 모바일 웹 또는 PWA로 시작한다. 설치 없이 접근할 수 있고, SNS 공유와 검색 유입에 유리하기 때문이다. 이후 기록, 알림, 구독 기능이 중요해지면 앱으로 확장한다.

## 2. 문제 정의

### 사용자가 겪는 문제
- 내 몸에 맞는 음식이나 생활 습관을 알고 싶지만 정보가 흩어져 있다.
- 사상체질, 8체질 관련 콘텐츠는 있지만 어렵거나 신뢰하기 어렵게 느껴진다.
- 테스트 결과를 봐도 실제 생활에서 무엇을 해야 할지 연결되지 않는다.
- 건강 정보는 많지만 나에게 맞춘 느낌이 부족하다.

### 시장의 빈틈
기존 서비스는 체질 테스트와 설명 제공에 머무르는 경우가 많다. 차별화 지점은 테스트 자체가 아니라, 결과를 바탕으로 매일 활용 가능한 개인화 가이드를 제공하는 데 있다.

## 3. 타겟 사용자

### 1차 타겟
20~40대 여성 중 건강, 다이어트, 자기이해 콘텐츠, MBTI/성향 테스트에 관심이 있는 사용자.

### 2차 타겟
소화, 피로, 몸에 맞는 음식, 계절별 컨디션 관리에 관심이 있는 사용자.

### 3차 타겟
한의학, 자연식, 건강 루틴, 체질식, 웰니스 콘텐츠에 관심이 있는 사용자.

### 초기 타겟 페르소나
이름: 지연, 32세 직장인

관심사:
- 자주 피곤하고 소화가 잘 안 되는 편이다.
- 다이어트와 건강식에 관심이 있다.
- MBTI, 퍼스널컬러, 체질 테스트처럼 나를 설명해주는 콘텐츠를 좋아한다.
- 병원에 갈 정도는 아니지만 내 몸에 맞는 생활 습관을 알고 싶다.

사용 동기:
- “나는 어떤 체질일까?”라는 호기심
- “내 체질에 맞는 음식은 뭘까?”라는 실용적 니즈
- 결과를 저장하거나 친구에게 공유하고 싶은 욕구

## 4. 제품 포지셔닝

### 권장 포지션
의료 진단 서비스가 아니라 생활 참고용 체질 웰니스 서비스로 포지셔닝한다.

### 피해야 할 표현
- 질병을 진단한다
- 치료한다
- 예방한다
- 특정 증상에 대한 의학적 판단을 제공한다
- 체질을 정확하게 판정한다

### 사용할 표현
- 생활 참고용
- 체질 경향
- 나에게 맞을 수 있는 음식
- 컨디션 관리 팁
- 일상 루틴 추천
- 정확한 진단은 전문가 상담 권장

## 5. 핵심 기능

### MVP 필수 기능
1. 사상체질 테스트
   - 태양인, 태음인, 소양인, 소음인 중 결과 제공
   - 문항은 너무 길지 않게 20~30문항 내외로 시작
   - 결과는 확정 진단이 아니라 경향성으로 안내

2. 결과 페이지
   - 체질명
   - 핵심 성향 요약
   - 몸/컨디션 특징
   - 잘 맞을 수 있는 음식
   - 주의하면 좋은 음식
   - 추천 생활 루틴
   - 공유용 결과 카드

3. 음식 가이드
   - 체질별 추천 음식
   - 체질별 주의 음식
   - 편의점/외식 메뉴 추천
   - 검색 가능한 음식 데이터 구조로 확장 가능하게 설계

4. 콘텐츠 페이지
   - 체질별 성격과 스트레스 반응
   - 체질별 운동 추천
   - 체질별 수면/피로 관리
   - 계절별 관리법
   - 다이어트 팁

5. 공유 기능
   - 카카오톡, 링크 복사, 이미지 저장 중심
   - 바이럴 유입을 위해 결과 카드의 디자인 완성도를 높임

### 2차 기능
- 오늘의 체질별 추천 음식
- 오늘의 컨디션 체크
- 먹은 음식 기록
- 음식이 나에게 맞았는지 피드백 기록
- 주간 컨디션 리포트
- 가족/커플 체질 궁합
- 8체질 확장 테스트
- 프리미엄 상세 리포트 결제

## 6. 8체질 포함 여부

### 권장 방향
초기 MVP에서는 사상체질 4가지로 시작한다. 8체질은 초기부터 핵심 기능으로 넣지 않고, 확장 콘텐츠 또는 프리미엄 콘텐츠 후보로 둔다.

### 이유
- 8체질은 설명 난도가 높고 콘텐츠 제작량이 많다.
- 정확성, 신뢰도, 표현 리스크 관리가 더 어렵다.
- 초기에는 사상체질만으로도 테스트, 결과, 음식, 루틴 콘텐츠를 충분히 만들 수 있다.
- 사용자의 관심도를 먼저 확인한 뒤 확장하는 편이 안전하다.

### 검증 방식
결과 페이지 또는 콘텐츠 하단에 “더 세분화된 8체질 테스트가 필요하신가요?” 버튼을 배치해 클릭률을 측정한다.

## 9. 수익모델

### 1차 수익모델: 프리미엄 리포트
무료 테스트 결과를 제공하고, 더 자세한 개인화 리포트를 유료로 판매한다.

예상 가격:
- 단건 상세 리포트: 3,900원~9,900원
- PDF 저장형 리포트: 5,900원~12,900원

리포트 구성:
- 체질 상세 설명
- 음식 가이드
- 7일 식단 예시
- 생활 루틴
- 다이어트/운동 팁
- 계절별 관리법

### 2차 수익모델: 구독
반복 사용 기능이 생긴 뒤 도입한다.

구독 기능 후보:
- 오늘의 음식 추천
- 주간 컨디션 리포트
- 체질별 루틴 알림
- 음식 기록 분석
- 프리미엄 콘텐츠 무제한 열람

### 3차 수익모델: 제휴
- 한의원 상담 연결
- 건강식품 제휴
- 도시락/식단 서비스 제휴
- 차/음료/건강 간식 브랜드 제휴

### 4차 수익모델: 광고
검색 유입이 충분히 커진 뒤 보조 수익으로 사용한다. 초기에는 사용자 경험을 해치기 쉬우므로 우선순위를 낮게 둔다.

## 12. 차별화 전략

### 단순 테스트 앱과 다른 점
- 결과를 예쁘게 보여주는 데서 끝나지 않고, 사용자가 오늘 바로 쓸 수 있는 음식과 루틴을 제공한다.
- 결과 공유를 고려한 콘텐츠 디자인을 만든다.
- 체질별 음식 데이터를 검색 가능하게 만든다.
- 8체질은 초기 과부하를 피하고, 사용자 수요가 확인되면 확장한다.

### 핵심 차별화 문장
“내 체질을 알고, 오늘 먹을 음식과 생활 루틴까지 바로 확인하는 체질 가이드.”

## 13. 리스크와 대응

### 정확성 리스크
자가 테스트만으로 체질을 확정하면 신뢰 문제가 생길 수 있다.

대응:
- 결과를 경향성으로 표현한다.
- 전문가 진단을 대체하지 않는다고 명확히 안내한다.
- 문항과 콘텐츠 출처를 관리한다.

### 의료 규제 리스크
진단, 치료, 예방 표현을 사용하면 의료 서비스로 오해될 수 있다.

대응:
- 생활 참고용 웰니스 서비스로 표현한다.
- 질병명 중심 콘텐츠는 피한다.
- 증상이 지속되면 전문가 상담을 권장한다.

### 콘텐츠 신뢰도 리스크
체질별 음식 정보는 출처마다 다를 수 있다.

대응:
- 내부 기준표를 만든다.
- 음식 추천은 절대 표현보다 “맞을 수 있음”, “주의해볼 수 있음”으로 안내한다.
- 사용자 피드백 데이터를 쌓아 개인화로 확장한다.

### 지속 사용 리스크
테스트 한 번 하고 이탈할 가능성이 높다.

대응:
- 음식 검색, 오늘의 추천, 컨디션 기록, 주간 리포트로 반복 사용 이유를 만든다.

## 15. 추천 첫 출시 버전

첫 출시 버전은 다음 흐름에 집중한다.

1. 사용자가 모바일 웹에 접속한다.
2. 사상체질 테스트를 시작한다.
3. 3분 이내에 결과를 확인한다.
4. 내 체질의 특징, 음식, 생활 루틴을 본다.
5. 결과 카드를 공유한다.
6. 더 자세한 리포트 또는 8체질 확장에 관심을 표현한다.

## 18. 최종 추천 방향

초기 제품은 “사상체질 테스트 기반 모바일 웹”으로 시작한다. 앱과 8체질은 나중에 확장한다. 핵심은 테스트 결과를 예쁘게 보여주는 것이 아니라, 사용자가 음식과 생활 루틴을 실제로 선택할 때 참고할 수 있게 만드는 것이다.

가장 좋은 MVP 콘셉트는 다음과 같다.

“내 체질을 테스트하고, 나에게 맞는 음식과 생활 루틴을 바로 확인하는 모바일 체질 가이드.”

## 19. 1차 MVP 확정안

### 제품 형태

1차 출시는 백엔드 없는 정적/프론트엔드 중심 반응형 웹으로 제작한다. 기본 경험은 모바일 웹에 최적화하고, PC에서도 자연스럽게 볼 수 있도록 반응형 레이아웃을 지원한다.

초기에는 회원가입, 로그인, 서버 저장, 결제, 관리자 페이지를 만들지 않는다. 테스트 답변과 결과 계산은 브라우저에서 처리하고, 사용자가 다시 볼 수 있는 최근 결과는 필요 시 브라우저의 localStorage에 저장한다.

### 핵심 화면

- 홈: 서비스 소개, 테스트 시작, 체질별 대표 키워드
- 테스트: 사상체질 문항, 진행률, 이전/다음 이동
- 결과: 사상체질 비율, 대표 체질, 성격 요약, 맞는 음식, 안 맞는 음식
- 체형 가이드: 태양인, 태음인, 소양인, 소음인별 남성/여성 체형 이미지와 설명
- 공유: 결과 링크 복사 또는 결과 카드 공유

### 결과 표현 방식

결과는 “귀하는 태음인입니다”처럼 단정하지 않고, “태음인 경향이 가장 높습니다” 또는 “태음인 62%, 소음인 24%, 소양인 10%, 태양인 4% 경향”처럼 비율형으로 제공한다.

PDF 진료지침에서도 사상체질 진단은 신체적 특성, 심리적 특성, 생리적/병리적 특성, 치료 반응 등을 함께 고려하는 과정으로 설명된다. 따라서 서비스에서는 자가 테스트 결과를 생활 참고용 경향으로 안내하고, 정확한 진단은 한의사 상담이 필요하다는 문구를 제공한다.

### 체형 이미지 정책

체형 콘텐츠는 남성 예시와 여성 예시를 모두 제공한다. 다만 성별에 따라 체질 자체가 달라진다고 표현하지 않는다. 성별은 체형 이미지와 설명을 더 직관적으로 보여주기 위한 시각화 기준으로만 사용한다.

체형 설명에는 “체형은 체질을 이해하는 참고 요소이며, 개인차가 큽니다”라는 문구를 함께 노출한다.

### 백엔드 도입 시점

초기에는 백엔드 없이 출시한다. 방문자 수, 테스트 완료율, 결과 공유율, 재방문율이 확인된 뒤 다음 기능이 필요해질 때 백엔드를 도입한다.

- 로그인
- 결과 저장 및 히스토리
- 음식/컨디션 기록
- 개인화 주간 리포트
- 유료 상세 리포트 결제
- 관리자 페이지
- 사용자 피드백 기반 추천

### 개인화 확장 방향

초기 빈틈은 백엔드가 아니라 결과 UX, 신뢰도 있는 콘텐츠, 체형 이미지, 음식 가이드의 완성도에서 만든다. 이후 사용자가 모이면 “내가 먹은 음식과 컨디션을 기록하고, 실제로 나에게 잘 맞는 음식을 찾아주는 개인화 가이드”로 확장한다.

이 확장 단계부터는 백엔드와 로그인 도입 가치가 커진다.

### 근거 자료
